# Phase 1a: MVP — Parse and Output

## Goal

`ariadne build <path>` works end-to-end. Parses a multi-language project via tree-sitter, outputs deterministic `graph.json` + `clusters.json`. Broken files are skipped with stderr warnings. No advanced features — just the core graph engine.

## Deliverables

### D1: Cargo Project (`ariadne-graph` crate)
- Binary name: `ariadne` (via `[[bin]]` in Cargo.toml, D-010)
- Dependencies: clap, tree-sitter + 7 grammar crates, serde/serde_json, xxhash-rust, walkdir, ignore, rayon, anyhow
- Dev-dependencies: insta (yaml), tempfile
- License: MIT OR Apache-2.0 (D-009)

### D2: Core Data Model (`src/graph/model.rs`)
- `Node`, `Edge`, `FileType`, `EdgeType`, `ArchLayer`, `ProjectGraph`, `Cluster`, `ClusterMap`
- `BTreeMap` for nodes and clusters (D-006 determinism)
- Edges sorted by (from, to, edge_type). All internal lists sorted.
- No `generated` timestamp in output.
- Per `architecture.md` Graph Data Model + `determinism.md`

### D3: Tree-sitter Integration
- Parse files into ASTs for import/export extraction
- Partial parse: if >50% top-level ERROR nodes → skip file (stderr warning). Otherwise extract from valid subtrees.
- Per `error-handling.md` W001/W007

### D4: LanguageParser Trait + Registry (`src/parser/mod.rs`)
- `LanguageParser` trait (Send + Sync, 6 methods), `Import`, `Export`, `ParserRegistry`
- Per `architecture.md` Language Support

### D5-D10: Tier 1 Language Parsers
- TypeScript/JavaScript (D5), Go (D6), Python (D7), Rust (D8), C# (D9), Java (D10)
- All import/export patterns per `architecture.md` language table
- Path resolution: relative paths → resolve, external/stdlib → skip
- No workspace-aware resolution in 1a (deferred to 1b)

### D11: File Type Detection (`src/detect/`)
- Source, Test, Config, Style, Asset, TypeDef — per `architecture.md` File types

### D12: Architectural Layer Inference (`src/detect/`)
- Api, Service, Data, Util, Component, Hook, Config, Unknown — directory-name heuristic

### D13: Content Hashing (`src/hash.rs`)
- xxHash64, lowercase hex (16 chars)

### D14: Graph Builder (`src/graph/mod.rs`)
- Pipeline: walk → read → parse → resolve → post-process → cluster → sort
- Walk via `ignore` crate (respects .gitignore)
- Parallel parsing via `rayon` on sorted file list
- Skip on: read error, not UTF-8, parse failure — log to stderr, continue
- Post-process: infer test edges, detect re-exports
- Sort everything for determinism (D-006)
- Per `architecture.md`, `determinism.md`, `performance.md`

### D15: JSON Serialization (`src/graph/serialize.rs`)
- graph.json: BTreeMap nodes, sorted edge tuples, version + project_root + node_count + edge_count
- clusters.json: BTreeMap clusters, sorted files
- Atomic writes (.tmp + rename)
- `BufWriter` + `serde_json::to_writer_pretty`
- Per `architecture.md` Storage Format, `determinism.md`, `performance.md`

### D16: Directory-based Clustering (`src/graph/cluster.rs`)
- Group by first meaningful directory segment. Cohesion metric. Sorted file lists.

### D17: CLI (`src/main.rs`)
- `ariadne build <path> [--output <dir>]` — default output: `.ariadne/graph/`
- `ariadne info` — version + supported languages
- Exit 0 on success (even with skipped files), exit 1 on fatal error (E001-E005 from `error-handling.md`)
- Summary to stdout: `"Built graph: N files, E edges, C clusters in Tms"`
- Skipped file count if any: `"X files skipped"`

### D18: Tests
- **L1 Parser snapshots** (`tests/parsers/`): ~44 tests via insta, one per import/export pattern
- **L2 Fixture graphs** (`tests/graph_tests.rs`): snapshot graph.json per fixture
- **L3 Invariants** (`tests/invariants.rs`): 13 structural invariants run on every fixture
- **L3 Determinism** test: build twice → assert byte-identical output
- **Fixtures:** typescript-app, go-service, python-package, mixed-project, edge-cases (empty, syntax error, circular imports)

### Deferred to Phase 1b
- Structured warning codes (W001-W009), JSON warning format
- CLI flags: --verbose, --warnings, --strict, --timestamp, --max-file-size, --max-files
- Workspace/monorepo detection (D-008)
- Case-insensitive FS handling, `dunce` crate
- Path normalization module (`src/resolve.rs`)
- Binary file detection (null byte check)
- Per-stage timing output
- Property-based tests (proptest), performance benchmarks (criterion)
- CI workflows, install.sh, README.md

## Success Criteria

1. `cargo build --release` compiles without errors
2. `ariadne info` lists all 6 Tier 1 languages
3. `ariadne build` on typescript-app fixture → correct graph.json with import edges
4. `ariadne build` on go-service fixture → correct graph.json
5. `ariadne build` on python-package fixture → correct graph.json
6. `ariadne build` on mixed-project fixture → files from all languages in one graph
7. edge-cases fixture: syntax error file skipped, circular imports produce edges, empty file is a node
8. graph.json format matches `architecture.md` (compact tuple edges, node metadata)
9. clusters.json has correct cohesion metrics
10. Output is byte-identical on repeated builds (determinism — D-006)
11. All `cargo test` pass
