# Phase 1a: MVP тАФ Parse and Output

## Goal

`ariadne build <path>` works end-to-end. Parses a multi-language project via tree-sitter, outputs deterministic `graph.json` + `clusters.json`. Broken files are skipped with stderr warnings. No advanced features тАФ just the core graph engine.

## Dependencies

None. Phase 1a is the first implementation phase.

## Deliverables

### D1: Cargo Project (`ariadne-graph` crate)
- Binary name: `ariadne` (via `[[bin]]` in Cargo.toml, D-010)
- Dependencies: clap, tree-sitter + grammar crates for 6 Tier 1 languages, serde/serde_json, xxhash-rust, ignore, rayon, thiserror (D-021)
- Dev-dependencies: insta (yaml), tempfile
- License: MIT OR Apache-2.0 (D-009)
- **Note:** `anyhow` removed тАФ `thiserror` for `FatalError`, concrete error types throughout (D-021)

### D2: Core Data Model (`src/model/`)
- **Newtypes (D-017):** `CanonicalPath`, `ContentHash`, `ClusterId`, `Symbol` in `src/model/types.rs`
- **`FileSet` (D-024):** `BTreeSet<CanonicalPath>` in `src/model/types.rs` тАФ used by `ImportResolver` for existence checks. Lives in `model/` so `parser/` can reference it without depending on `pipeline/`.
- **Internal model:** `Node`, `Edge`, `FileType`, `EdgeType`, `ArchLayer`, `ProjectGraph`, `ClusterMap` using newtypes
- **Output model (D-022):** `GraphOutput`, `NodeOutput`, `ClusterOutput` in `src/serial/mod.rs` тАФ string keys, compact tuple edges (D-012)
- `BTreeMap<CanonicalPath, Node>` for nodes, `BTreeMap<ClusterId, Cluster>` for clusters (D-006)
- Conversion: `impl From<ProjectGraph> for GraphOutput` тАФ all sort-point enforcement here
- Enums derive `Copy + Ord` with `#[serde(rename_all = "snake_case")]`
- No `generated` timestamp in output.
- Per `architecture.md` Graph Data Model + `determinism.md`

### D3: Tree-sitter Integration
- Parse files into ASTs for import/export extraction
- Partial parse: if >50% top-level ERROR nodes тЖТ skip file (DiagnosticCollector warning). Otherwise extract from valid subtrees.
- Per `error-handling.md` W001/W007

### D4: Traits + Registry (`src/parser/`)
- **`LanguageParser` trait** (Send + Sync, 5 methods): `language`, `extensions`, `tree_sitter_language`, `extract_imports`, `extract_exports` (D-018)
- **`ImportResolver` trait** (Send + Sync, 1 method): `resolve(import, from_file, known_files: &FileSet) -> Option<CanonicalPath>` (D-018)
- **Intermediate types:** `RawImport`, `RawExport` (from parser), `FileEntry`, `FileContent`, `ParsedFile` (between pipeline stages)
- `ParserRegistry` with register/lookup/`with_tier1()`. `lookup` returns `Option` for unknown extensions (unrecognized files silently skipped).
- Per `architecture.md` Language Support + Language-Specific Resolution Notes

### D5: Pipeline Traits + Support Types (`src/pipeline/`, D-024)
- **`FileWalker` trait** + `FsWalker` impl (D-019): walk via `ignore` crate, always excludes `.ariadne/`
- **`FileReader` trait** + `FsReader` impl (D-019): read + hash + filter, returns `Result<FileContent, FileSkipReason>`
- **`BuildPipeline` struct**: accepts `Box<dyn FileWalker>`, `Box<dyn FileReader>`, `ParserRegistry`, `Box<dyn GraphSerializer>` (D-019)
- **`DiagnosticCollector`**: thread-safe `Mutex<Vec<Warning>>`, sorted drain (D-021)
- **Pipeline support types (D-024):** `FileSkipReason` (enum), `WalkConfig`, `BuildOutput` тАФ per `architecture.md` Pipeline Support Types
- **Note:** `FileSkipReason::BinaryFile` variant is defined for type completeness but not emitted in Phase 1a тАФ null-byte detection logic is deferred to Phase 1b.
- `pipeline.run(root, config) -> Result<BuildOutput, FatalError>`
- Per `architecture.md` Pipeline Architecture

### D6-D11: Tier 1 Language Parsers
- TypeScript/JavaScript (D6), Go (D7), Python (D8), Rust (D9), C# (D10), Java (D11)
- Each implements `LanguageParser` + `ImportResolver` (can be one struct, two impls)
- Returns `RawImport`/`RawExport` from parser, resolution via `ImportResolver`
- No workspace-aware resolution in 1a (deferred to 1b)
- Per `architecture.md` Language Support (Tier 1 table), Language-Specific Resolution Notes (Rust module path mapping), TypeScript/JavaScript resolution (Phase 1a)

### D12: File Type Detection (`src/detect/filetype.rs`)
- Source, Test, Config, Style, Asset, TypeDef тАФ per `architecture.md` File Type Detection (6-level priority table, per-language test patterns)
- Filename-specific rules take precedence over extension rules (e.g., `tsconfig.json` тЖТ config, not asset)
- Returns `FileType` enum

### D13: Architectural Layer Inference (`src/detect/layer.rs`)
- Api, Service, Data, Util, Component, Hook, Config, Unknown тАФ per `architecture.md` Architectural Layer Heuristics (directory pattern table)
- Covers both frontend (component, hook) and backend (controllers, handlers, repositories) conventions
- `unknown` is expected to be common for backend-only projects
- Returns `ArchLayer` enum

### D14: Content Hashing (`src/hash.rs`)
- xxHash64, returns `ContentHash` newtype, lowercase hex (16 chars) (D-013)

### D15: Graph Builder (`src/pipeline/build.rs`)
- Pipeline: walk тЖТ read тЖТ parse тЖТ resolve_and_build тЖТ cluster тЖТ convert + serialize
- Parallel parsing via `rayon` on sorted file list (preserves order)
- `FileSet` built from successfully-read files only (prevents dangling edge targets from TOCTOU)
- Skip on: read error, not UTF-8, parse failure тАФ via `DiagnosticCollector`, continue
- resolve_and_build sub-responsibilities (per `architecture.md`):
  1. Build `FileSet` from read results
  2. Detect `FileType` and `ArchLayer` per file via `detect/`
  3. Resolve imports via `ImportResolver`
  4. Classify edges: `tests`, `re_exports`, `type_imports`, or `imports` (per `architecture.md` Edge Type Inference)
  5. Apply naming-convention test edge inference
  6. Deduplicate edges: same (from, to, edge_type) тЖТ merge symbols (union, sorted)
  7. `arch_depth` = 0 for all nodes (D-025, computed in Phase 2)
  8. Sort everything for determinism (D-006)
- `ClusterMap тЖТ ClusterOutput` conversion happens in `pipeline/`, not `serial/` (preserves `serial/` dependency rules)
- Per `architecture.md` (resolve_and_build sub-responsibilities), `determinism.md`, `performance.md`

### D16: JSON Serialization (`src/serial/json.rs`)
- **`GraphSerializer` trait** + `JsonSerializer` impl
- graph.json: via `GraphOutput` тАФ BTreeMap string keys, sorted compact tuple edges (D-012)
- clusters.json: via `ClusterOutput` тАФ BTreeMap string keys, sorted files
- Atomic writes (.tmp + rename)
- `BufWriter` + `serde_json::to_writer_pretty`
- Per `architecture.md` Storage Format, `determinism.md`, `performance.md`

### D17: Directory-based Clustering (`src/cluster/mod.rs`)
- `assign_clusters(graph: &ProjectGraph) -> ClusterMap` тАФ returns data, does NOT mutate graph
- Group by first meaningful directory segment. Cohesion metric (rounded to 4 decimal places per `determinism.md`). Sorted file lists.
- Returns `ClusterMap` with `ClusterId` keys.
- Per `architecture.md` Cluster Interface

### D18: CLI + Composition Root (`src/main.rs`)
- Composition Root (D-020): sole place where concrete types are wired
- `ariadne build <path> [--output <dir>]` тАФ default output: `.ariadne/graph/`
- `ariadne info` тАФ version + supported languages
- Exit 0 on success (even with skipped files), exit 1 on `FatalError` (E001-E005)
- Summary to stdout: `"Built graph: N files, E edges, C clusters in Tms"`
- Skipped file count if any: `"X files skipped"` (part of summary block)
- Per `architecture.md` CLI Interface, `error-handling.md` Summary Report

### D19: Tests
- **L1 Parser snapshots** (`tests/parsers/`): ~50 tests via insta, one per import/export pattern + ~20 path resolution tests тАФ per `testing.md` Coverage Matrix
- **Pipeline unit tests**: mock-based tests with `MockWalker`/`MockReader` (D-019)
- **L2 Fixture graphs** (`tests/graph_tests.rs`): snapshot graph.json per fixture
- **L3 Basic invariants** (`tests/invariants.rs`): INV-1 (edge referential integrity), INV-2 (no self-imports), INV-8 (counts match), INV-9 (no duplicates), INV-11 (determinism тАФ byte-identical). Full 13-invariant suite with proptest in Phase 1b.
- **Fixtures (8):** typescript-app, go-service, python-package, rust-crate, csharp-project, java-project, mixed-project, edge-cases (empty, syntax error, circular imports)

### Deferred to Phase 1b
- User-facing warning output: JSON warning format (`--warnings json`), `--verbose` flag (gates W006 display), `--strict` flag. **Note:** internal warning types (`WarningCode`, `Warning`, `DiagnosticCollector`) are Phase 1a тАФ only the CLI surface and formatted output are deferred.
- CLI flags: --timestamp, --max-file-size, --max-files (underlying `WalkConfig` fields exist in Phase 1a with defaults)
- Workspace/monorepo detection (D-008)
- Case-insensitive FS handling, `dunce` crate
- Binary file detection (null byte check) тАФ `FileSkipReason::BinaryFile` type exists but is not emitted
- Per-stage timing output
- Full L3 invariant suite (INV-3 through INV-7, INV-10, INV-12, INV-13) with proptest
- Performance benchmarks (criterion)
- CI workflows, install.sh, README.md

## Success Criteria

1. `cargo build --release` compiles without errors
2. `ariadne info` lists all 6 Tier 1 languages
3. `ariadne build` on typescript-app fixture тЖТ correct graph.json with import edges
4. `ariadne build` on go-service fixture тЖТ correct graph.json
5. `ariadne build` on python-package fixture тЖТ correct graph.json
6. `ariadne build` on rust-crate fixture тЖТ correct graph.json
7. `ariadne build` on mixed-project fixture тЖТ files from all languages in one graph
8. edge-cases fixture: syntax error file skipped, circular imports produce edges, empty file is a node
9. graph.json format matches `architecture.md` (compact tuple edges, node metadata)
10. clusters.json has correct cohesion metrics
11. Output is byte-identical on repeated builds (determinism тАФ D-006)
12. All `cargo test` pass
