# Releasing Ariadne

Ariadne ships a single binary, `ariadne` (from `ariadne-cli`), via
[cargo-dist](https://opensource.axo.dev/cargo-dist/) v0.31.0. A pushed `vX.Y.Z`
tag triggers `.github/workflows/release.yml`, which builds per-target archives
for macOS (arm64 + x64), Windows (x64), and Linux (arm64 + x64), generates the
`shell`, `powershell`, and `homebrew` installers, creates a GitHub Release with
those assets, and publishes the Homebrew formula to a tap.

`release.yml` is autogenerated from `dist-workspace.toml` — never hand-edit it.
After changing dist config, run `dist generate` and confirm `dist generate
--check` exits 0.

## One-time setup (manual)

The Homebrew tap publish job needs a repo and a token that no script in this
project can create. Do these once, by hand:

1. **Create the tap repo.** Create a new GitHub repository
   `mind-decay/homebrew-tap` (the `homebrew-` prefix is the tap naming
   convention). Initialize it with a README so the default branch exists — the
   publish job checks the repo out and commits the formula to it. The repo must
   already exist before the first release; dist does not create it.

2. **Create a Personal Access Token.** Create a *classic* PAT with the `repo`
   scope. This token must be able to push to `mind-decay/homebrew-tap`. (A
   fine-grained token with contents read/write on the tap repo also works; the
   classic `repo` scope is the documented path.)

3. **Add the token as a secret.** In the **`mind-decay/ariadne`** repository (the
   source repo, not the tap), add the PAT as an Actions secret named
   `HOMEBREW_TAP_TOKEN`. The `publish-homebrew-formula` job reads
   `secrets.HOMEBREW_TAP_TOKEN` to push the formula to the tap.

## Cutting a release

1. Bump the workspace version in `Cargo.toml` (`[workspace.package] version`).
2. Regenerate the changelog: `cog changelog` (see `cog.toml`).
3. Commit the version bump + changelog with a conventional commit.
4. Tag the release: `git tag vX.Y.Z` (the tag must match the bumped version).
5. Push the tag: `git push origin vX.Y.Z`.

The pushed tag starts the Release workflow. On success the GitHub Release
carries the archives and the `shell` / `powershell` installer scripts, and the
`publish-homebrew-formula` job commits `ariadne-cli.rb` to `mind-decay/homebrew-tap`.

## Installing a released build

- Homebrew: `brew install mind-decay/homebrew-tap/ariadne-cli`
- Shell (macOS/Linux): `curl --proto '=https' --tlsv1.2 -LsSf <release-url>/ariadne-cli-installer.sh | sh`
- PowerShell (Windows): `irm <release-url>/ariadne-cli-installer.ps1 | iex`

## Licensing note for the tap

Ariadne is licensed under PolyForm Noncommercial 1.0.0 — a source-available,
non-OSI license. Homebrew-core requires an OSI-approved license, but that
requirement applies only to the official `homebrew/core` tap. A **self-hosted
tap** (`mind-decay/homebrew-tap`) has no such constraint, so the PolyForm
license is fine here. See
[How to Create and Maintain a Tap](https://docs.brew.sh/How-to-Create-and-Maintain-a-Tap).

## Verifying config without a release

- `dist plan` lists the installer artifacts (`ariadne-cli-installer.sh`,
  `ariadne-cli-installer.ps1`, `ariadne-cli.rb`) and the per-target archives.
- `dist generate --check` confirms `release.yml` matches `dist-workspace.toml`.

Full proof requires the tap repo + `HOMEBREW_TAP_TOKEN` secret and a real `v*`
tag: the published Release assets and the formula appearing in
`mind-decay/homebrew-tap`.
