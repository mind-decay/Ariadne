<context>
`dist-workspace.toml` currently sets `installers = []`, so releases ship raw
archives only [src: dist-workspace.toml:18]. Targets already cover macOS
arm64+x64 and Windows x64 [src: dist-workspace.toml:20]. Add the three installers
the owner chose and a Homebrew tap, then regenerate the CI workflow `dist` owns.
Depends on tier-01: the Homebrew formula embeds the Cargo `license`, so the
PolyForm id must be in place first [src: plan.md D3, D4].
</context>

<files>
- `dist-workspace.toml` — modify; populate `installers` and add `tap`.
- `.github/workflows/release.yml` — regenerated by `dist`; do not hand-edit.
- `docs/releasing.md` — create; tap repo + secret + tag-to-release runbook.
</files>

<steps>
1. Ensure the pinned `dist` is available: `dist --version` should report 0.31.0;
   if absent install it with the pinned script
   `curl --proto '=https' --tlsv1.2 -LsSf https://github.com/axodotdev/cargo-dist/releases/download/v0.31.0/cargo-dist-installer.sh | sh`
   [src: .github/workflows/release.yml:67].
2. In `dist-workspace.toml` set
   `installers = ["shell", "powershell", "homebrew"]` and
   `tap = "mind-decay/homebrew-tap"`. `installers` accepts exactly these strings;
   `tap` names the GitHub repo dist publishes the formula to (convention:
   `homebrew-*`) [src:
   https://raw.githubusercontent.com/axodotdev/cargo-dist/main/book/src/reference/config.md].
3. Regenerate CI: run `dist generate` (rewrites `.github/workflows/release.yml`
   from config). Then `dist generate --check` must report no drift. Do not edit
   `release.yml` by hand — it is autogenerated [src: release.yml:1-3].
4. Write `docs/releasing.md` documenting the one-time manual setup (a script
   cannot create these): (a) create repo `mind-decay/homebrew-tap`, initialized
   with a README; (b) create a classic PAT with `repo` scope; (c) add it to the
   `mind-decay/ariadne` repo as Actions secret `HOMEBREW_TAP_TOKEN`; (d) the
   release flow: bump version, `cog changelog`, tag `vX.Y.Z`, push tag — the
   Release workflow builds artifacts and publishes the formula [src:
   https://raw.githubusercontent.com/axodotdev/cargo-dist/main/book/src/installers/homebrew.md].
5. Note in `docs/releasing.md` that the PolyForm (non-OSI) license is fine for a
   personal tap — Homebrew-core's OSI requirement does not apply to a
   self-hosted tap [src: https://docs.brew.sh/How-to-Create-and-Maintain-a-Tap].
</steps>

<verification>
- `grep -A1 installers dist-workspace.toml` shows the three installers; `tap` set.
- `dist plan --output-format=json | jq '.artifacts | keys'` (or `dist plan`
  human output) lists `shell`, `powershell`, and `homebrew` installer artifacts,
  and per-target archives for `aarch64-apple-darwin`, `x86_64-apple-darwin`,
  `x86_64-pc-windows-msvc`.
- `dist generate --check` → exits 0 (no uncommitted CI drift).
- `docs/releasing.md` exists and names `homebrew-tap`, `repo` scope,
  `HOMEBREW_TAP_TOKEN`.
- Full proof (owner-run, needs the tap repo + secret + a real tag): pushing a
  `v*` tag yields a GitHub Release whose assets include the installer scripts and
  a formula appears in `mind-decay/homebrew-tap`.
</verification>

</rollback>
