<context>
v1 `SymbolRecord` carries only `canonical_name`, `kind`, `defining_file`, `defining_span` [src: crates/ariadne-core/src/domain/records.rs:28-37]. The tier-05 dead-code classifier (plan RD4) cannot see Rust `pub`/`#[test]`, JS/TS exports, Java/C# annotations, or Python `__main__` — only `main`-by-name and Go `Test*` by convention. This prerequisite tier enriches `SymbolRecord` with visibility + attributes (plan RD10) so the classifier reads observable metadata, not name heuristics. Full context: plan.md.
</context>

<steps>
1. Failing test first (`crates/ariadne-storage/tests/migration.rs`): a `#[test]` helper writes a redb file at `SCHEMA_VERSION = 2` with known `SymbolRecord`s; bump the in-code constant to 3; reopen → assert the migration runs, every record's `canonical_name`/`kind`/`defining_file`/`defining_span` is byte-identical, `visibility == Visibility::Unknown`, `attributes` empty. Red — no v2->v3 step exists [src: crates/ariadne-storage/tests/migration.rs, tier-02 round-trip pattern].
2. ariadne-core: add `Visibility { Public, Restricted, Private, Unknown }` in `domain/types/visibility.rs` — `#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]`, `Default = Unknown`; re-export through `types/mod.rs` and the `lib.rs` façade, mirroring `Lang` [src: crates/ariadne-core/src/domain/types/lang.rs; CLAUDE.md conventions — adapter façade re-exports only].
3. ariadne-core: extend `SymbolRecord` with `pub visibility: Visibility` and `pub attributes: Vec<String>`, appended after `defining_span` so the v3 postcard layout extends the v2 byte prefix [src: crates/ariadne-core/src/domain/records.rs:28-37].
4. ariadne-storage migration: postcard is non-self-describing — struct field count/names are not on the wire, so adding fields breaks decode of v2 bytes [src: https://postcard.jamesmunns.com/wire-format]. Define a private frozen `SymbolRecordV2 { canonical_name, kind, defining_file, defining_span }` in `migration.rs`; `migrate_v2_to_v3` opens the `SYMBOLS` table, collects each `(key, decode_value::<SymbolRecordV2>(value))`, then re-inserts each key with `encode_value` of a v3 `SymbolRecord` whose `visibility = Visibility::Unknown` and `attributes = Vec::new()` [src: crates/ariadne-storage/src/domain/migration.rs:22-52; crates/ariadne-storage/src/adapters/codec.rs:26-34].
5. Register the step: `MigrationRegistry::builtin` appends `MigrationStep { from: 2, to: 3, apply: migrate_v2_to_v3 }`; bump `SCHEMA_VERSION` `2` -> `3` in `tables.rs`. The tier-02 runner already wraps the chain in one `WriteTransaction` and owns the `meta` `schema_version` key [src: crates/ariadne-storage/src/domain/migration.rs:19-21,44-52; crates/ariadne-storage/src/adapters/redb/tables.rs:8].
6. ariadne-parser: add `pub visibility: Visibility` + `pub attributes: Vec<String>` to `Decl` [src: crates/ariadne-parser/src/adapters/treesitter/facts.rs:86-97]. Extend each base-language `.scm` with `@visibility` on the visibility-modifier node and `@attribute` on attribute/annotation nodes — Rust `visibility_modifier` + `attribute_item`; Java/Kotlin `modifiers` + `annotation`/`marker_annotation`; C# `modifier` + `attribute_list`; TS/JS/TSX `export_statement` + `decorator` + `accessibility_modifier`; Python `decorator`; C/C++ storage-class specifier + C++ `access_specifier` + `attribute_declaration` [src: tree-sitter query captures, https://tree-sitter.github.io/tree-sitter/using-parsers/queries/1-syntax.html]. Verify every capture against the grammar's `node-types.json`.
7. ariadne-parser `FactExtractor::extract`: map `@visibility`/`@attribute` captures onto `Decl`; Go has no visibility keyword and no annotations — derive `visibility` from the leading-character case of the decl name (exported = uppercase) [src: https://go.dev/ref/spec#Exported_identifiers]. Where a grammar exposes neither signal, emit `Visibility::Unknown` + empty `attributes` and record the gap in the tier audit [src: crates/ariadne-parser/src/adapters/treesitter/facts.rs:286-378].
8. ariadne-scip: SCIP `SymbolInformation` has no visibility or attribute field; a symbol string of the form `local <local-id>` is Document-local, any other (descriptor) form is package-visible [src: crates/ariadne-scip/proto/scip.proto:145-191,248-266]. Map `local` -> `Visibility::Private`, descriptor form -> `Visibility::Public`; leave `attributes` empty. ADR-0014 records this as a known SCIP limitation.
9. ariadne-cli ingest: in the `SymbolRecord` construction populate `visibility` + `attributes` from the parser `Decl` (syntactic path) or the SCIP mapping (semantic path) [src: crates/ariadne-cli/src/domain/mod.rs:550-561].
10. ariadne-salsa: add `visibility` + `attributes` to `SymbolFactsRaw` and `DeclRaw`; `Visibility` is `Copy` and `Vec<String>` is `salsa::Update`-safe — confirm the derive holds [src: crates/ariadne-salsa/src/derived.rs:35-46,66-79].
11. Write `docs/adr/0014-symbol-metadata-enrichment.md` (template): decision = enrich `SymbolRecord`; migration = re-encode `SYMBOLS` v2->v3 via the tier-02 `MigrationRegistry`; SCIP visibility-only limitation. Status `Accepted`; cite from plan RD10. Run the full verification; step 1's test goes green.
</steps>
