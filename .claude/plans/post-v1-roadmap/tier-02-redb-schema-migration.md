<context>
v1 stores a schema version in redb and, on mismatch, returns `SchemaMismatch` so the caller rebuilds the whole index from source — discarding SCIP ingest cost on every format bump (tier-02 hard limit; test `reopen_with_mismatched_schema_version_returns_schema_mismatch` in `crates/ariadne-storage/tests/changeset.rs`). This tier adds an in-place migration framework (plan RD2). No ADR — RD2 records the decision; this is an enhancement within the existing storage adapter, not a new pattern. Full context: plan.md.
</context>

<files>
- crates/ariadne-storage/src/domain/migration.rs — new: `MigrationStep` (pure `from`/`to`/transform) + `MigrationRegistry` (ordered chain, planner).
- crates/ariadne-storage/src/adapters/redb.rs — modify the open path: version < current → run the migration chain instead of erroring; version > current or no path → keep `SchemaMismatch`.
- crates/ariadne-storage/src/errors.rs — modify: add a `Migration` variant to the storage error enum.
- crates/ariadne-storage/tests/migration.rs — new: round-trip test.
- crates/ariadne-storage/fixtures/ — new: a redb file written at the previous schema version (generated by a `#[test]` helper, committed).
</files>

<steps>
1. Failing test first (`tests/migration.rs`): write a redb file at `current-1` with known records, bump the in-code schema constant, reopen → assert it migrates and every record is byte-identical. Red — no migration path exists.
2. Read the redb open + schema-version code in `redb.rs` and the `SchemaMismatch` path to locate the version constant and table set [src: crates/ariadne-storage/src/adapters/redb.rs].
3. Define `MigrationStep { from: u32, to: u32, apply: fn(&WriteTransaction) -> Result<(), StorageError> }` and `MigrationRegistry` holding an ordered `Vec<MigrationStep>`; a `plan(from, to)` returns the contiguous step chain or `None`.
4. All steps for one open run inside a single redb `WriteTransaction` so a mid-migration crash leaves the file at the original version (ACID — partial migration never commits) [src: https://docs.rs/redb/4.1.0/redb/struct.WriteTransaction.html].
5. Migrated open path: read on-disk version; if < current and `plan` is `Some`, apply the chain, write the new version, commit; if `None` or version > current, return `SchemaMismatch` unchanged.
6. Register one real identity/no-op step `current-1 -> current` so the framework is exercised end-to-end with a registered path (future format bumps append steps).
7. Add the `Migration` error variant; keep `thiserror` (no `anyhow` in an adapter crate).
</steps>

- `cargo test --test architecture`, `cargo clippy --workspace --all-targets -- -D warnings`, `cargo fmt --all --check` — green.
</verification>
