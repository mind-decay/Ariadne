<context>
Final tier. Glues every prior tier behind a single binary, then proves SLOs against real codebases (not synthetic). This is the v1 release gate — if any SLO fails, v1 does not ship.
</context>

<files>
- crates/ariadne-cli/Cargo.toml — `clap` (derive), `toml`, `directories`, workspace deps + every internal crate.
- crates/ariadne-cli/src/main.rs — clap derive subcommand dispatch.
- crates/ariadne-cli/src/commands/{init,index,watch,serve,query,status,mem}.rs — per-subcommand impl.
- crates/ariadne-cli/src/config.rs — load + validate `.ariadne/config.toml`; merge env overrides.
- crates/ariadne-e2e/Cargo.toml — `assert_cmd`, `tempfile`, `git2` (for shallow clone), tokio.
- crates/ariadne-e2e/tests/repos/{rust,typescript,python,go,java,csharp}.rs — one suite per lang.
- crates/ariadne-e2e/fixtures/repos.toml — list of pinned commits per fixture repo (e.g., `microsoft/vscode @ <SHA>`, subset of `torvalds/linux`, etc.).
- crates/ariadne-e2e/tests/slo.rs — combined SLO bench (cold + incremental + query p95).
- crates/ariadne-e2e/tests/mcp_session.rs — spawns `ariadne mcp` and replays a realistic Claude Code session (initialize, tools/list, 50 random tools/call).
- .github/workflows/release.yml — `cargo dist` release pipeline.
- README.md (repo root) — install, quickstart, mcp.json, Go install hint, troubleshooting.
- docs/architecture.md (auto-generated by tier-09 doc_for_project; committed snapshot).
</files>

<steps>
1. **Failing test first** (tests/slo.rs): clone a 100K-file fixture into tempdir, run `ariadne index --root <tmp>`, assert wall-clock < 60s; then mutate 1 file, await `ariadne watch` invalidation, assert latency < 500ms; then issue 100 random `blast_radius` queries, assert p95 < 100ms. Fails until full pipeline integrated.
2. clap CLI scaffold (src/main.rs): `#[derive(Parser)] enum Cmd { Init { root: PathBuf }, Index { root: PathBuf, fresh: bool }, Watch { root: PathBuf }, Serve { root: PathBuf }, Query { tool: String, args_json: String, root: PathBuf }, Status { root: PathBuf }, Mem { root: PathBuf } }`.
3. `init`: create `.ariadne/` dir, write default `config.toml` (enabled_langs autodetected from repo signals, ignore=defaults+.gitignore-respect=true, indexer paths from PATH lookup), add `.ariadne/` to `.gitignore` (idempotent).
4. `index`: orchestrate full cold pipeline: WalkBuilder → parser facts → SCIP ingest (parallel) → AriadneDb seed → graph build → commit to redb. Print progress via `indicatif`; emit JSON-line summary on stdout.
5. `watch`: open AriadneDb, start tier-06 Watcher, log invalidations + apply latencies, idle forever; SIGINT clean shutdown.
6. `serve`: thin wrapper around `ariadne-mcp serve` (tier-08) plus optional `--watch` flag that runs Watcher in same process.
7. `query <tool> <args_json>`: in-process call to AriadneServer tool router; useful for shell/CI debugging without MCP handshake. Prints JSON to stdout.
8. `status`: print revision, file count, symbol count, edge count, last index timestamp, configured langs, indexer availability matrix.
9. `mem`: print `AriadneDb::memory_report()` per table; flag any > 256MB with a warning + suggestion.
10. E2E suites: for each lang/repo (rust=ariadne_v2 self + tokio; ts=microsoft/vscode subset; py=pandas; go=kubernetes subset; java=spring-projects/spring-boot subset; c#=dotnet/runtime subset), shallow-clone at pinned commit, run `ariadne index`, assert non-empty graph, SLO check. Pin commit SHAs so future repo changes do not silently break tests.
10a. Real-repo per-lang SCIP goldens (carries the tier-05 deferral): on a CI image bundling all 7 indexers, index one ≤100-LOC fixture repo per lang via its actual `ScipIndexer::run`; insta snapshots `IngestReport` + symbol-count + relationship-count + top-5 symbols by occurrence. Failure surfaces drift between cached fixture and live indexer output.
10b. Manual per-lang ingest walk (carries tier-05 `<verification>` bullet 2): on each fixture lang repo, run `ariadne ingest-scip` and confirm `IngestReport.successes` carries that lang. Go path verification (carries tier-05 `<verification>` bullet 3): index `golang/example` via lsif-go + `scip convert` → assert non-empty `ScipDoc` with `Definition` + `Reference` occurrences. Document outcome in tier-10 audit report.
11. tests/mcp_session.rs: scripted MCP transcript — spawn `ariadne serve`, send 50 realistic JSON-RPC messages, assert each response within 100ms p95 and no error frames.
12. cargo-dist release config (`Cargo.toml [workspace.metadata.dist]`): targets x86_64/aarch64 × linux/macos/windows; checksums + GH release artifact upload [src: https://opensource.axo.dev/cargo-dist/].
13. README sections: Why → Install → Quickstart (`ariadne init && ariadne index && add to mcp.json`) → Tool catalog (links to per-tool docs) → Indexer install matrix (per-lang command + min version) → Troubleshooting (memory probe, watcher diagnostics, Go fallback).
14. Run tier-09 `docgen::for_project()` on the ariadne_v2 self-index, commit the rendered Markdown as `docs/architecture.md`. Repeat each release.
15. Manual end-to-end smoke before declaring v1 done: add ariadne to a fresh Claude Code project, run `/mcp`, ask Claude "explain the blast radius of WriteTxn::apply_changeset" — compare answer against tier-07 golden output. Document outcome in tier-10 audit report.
</steps>
