<steps>
1. **Failing test first** (tests/slo.rs): clone a 100K-file fixture into tempdir, run `ariadne index --root <tmp>`, assert wall-clock < 60s; then mutate 1 file, await `ariadne watch` invalidation, assert latency < 500ms; then issue 100 random `blast_radius` queries, assert p95 < 100ms. Fails until full pipeline integrated.
2. clap CLI scaffold (src/main.rs): `#[derive(Parser)] enum Cmd { Init { root: PathBuf }, Index { root: PathBuf, fresh: bool }, Watch { root: PathBuf }, Serve { root: PathBuf }, Query { tool: String, args_json: String, root: PathBuf }, Status { root: PathBuf }, Mem { root: PathBuf } }`.
3. `init`: create `.ariadne/` dir, write default `config.toml` (enabled_langs autodetected from repo signals, ignore=defaults+.gitignore-respect=true, indexer paths from PATH lookup), add `.ariadne/` to `.gitignore` (idempotent).
4. `index`: orchestrate full cold pipeline: WalkBuilder ‚Üí parser facts ‚Üí SCIP ingest (parallel) ‚Üí AriadneDb seed ‚Üí graph build ‚Üí commit to redb. Print progress via `indicatif`; emit JSON-line summary on stdout.
5. `watch`: open AriadneDb, start tier-06 Watcher, log invalidations + apply latencies, idle forever; SIGINT clean shutdown.
6. `serve`: thin wrapper around `ariadne-mcp serve` (tier-08) plus optional `--watch` flag that runs Watcher in same process.
7. `query <tool> <args_json>`: in-process call to AriadneServer tool router; useful for shell/CI debugging without MCP handshake. Prints JSON to stdout.
8. `status`: print revision, file count, symbol count, edge count, last index timestamp, configured langs, indexer availability matrix.
9. `mem`: print `AriadneDb::memory_report()` per table; flag any > 256MB with a warning + suggestion.
10. E2E suites: for each lang/repo (rust=ariadne_v2 self + tokio; ts=microsoft/vscode subset; py=pandas; go=kubernetes subset; java=spring-projects/spring-boot subset; c#=dotnet/runtime subset), shallow-clone at pinned commit, run `ariadne index`, assert non-empty graph, SLO check. Pin commit SHAs so future repo changes do not silently break tests.
10a. Real-repo per-lang SCIP goldens (carries the tier-05 deferral): on a CI image bundling all 7 indexers, index one ‚â§100-LOC fixture repo per lang via its actual `ScipIndexer::run`; insta snapshots `IngestReport` + symbol-count + relationship-count + top-5 symbols by occurrence. Failure surfaces drift between cached fixture and live indexer output.
10b. Manual per-lang ingest walk (carries tier-05 `<verification>` bullet 2): on each fixture lang repo, run `ariadne ingest-scip` and confirm `IngestReport.successes` carries that lang. Go path verification (carries tier-05 `<verification>` bullet 3): index `golang/example` via lsif-go + `scip convert` ‚Üí assert non-empty `ScipDoc` with `Definition` + `Reference` occurrences. Document outcome in tier-10 audit report.
11. tests/mcp_session.rs: scripted MCP transcript ‚Äî spawn `ariadne serve`, send 50 realistic JSON-RPC messages, assert each response within 100ms p95 and no error frames.
12. cargo-dist release config (`Cargo.toml [workspace.metadata.dist]`): targets x86_64/aarch64 √ó linux/macos/windows; checksums + GH release artifact upload [src: https://opensource.axo.dev/cargo-dist/].
13. README sections: Why ‚Üí Install ‚Üí Quickstart (`ariadne init && ariadne index && add to mcp.json`) ‚Üí Tool catalog (links to per-tool docs) ‚Üí Indexer install matrix (per-lang command + min version) ‚Üí Troubleshooting (memory probe, watcher diagnostics, Go fallback).
14. Run tier-09 `docgen::for_project()` on the ariadne_v2 self-index, commit the rendered Markdown as `docs/architecture.md`. Repeat each release.
15. Manual end-to-end smoke before declaring v1 done: add ariadne to a fresh Claude Code project, run `/mcp`, ask Claude "explain the blast radius of WriteTxn::apply_changeset" ‚Äî compare answer against tier-07 golden output. Document outcome in tier-10 audit report.
</steps>
